What They Mean

Tech concepts for non-developers — one idea per screen, shown rather than told.

What is an API?

Every pretty app is really just data, drawn nicely — peek behind one and watch the request fetch the raw JSON underneath.

Learn more

An API is a waiter between programs. Your app asks for something — "today's weather", "this user's orders" — and the API brings back the raw facts as text. The screen you see is just those facts dressed up; change the data and the screen changes.

What is a database?

A spreadsheet many machines can share at once — flip between its sheets and ask it precise questions.

Learn more

A database is a giant shared spreadsheet that thousands of people and programs can read and write at the same time without stepping on each other. Instead of scrolling, you ask it precise questions — "all orders over $50 from June" — and it hands back just those rows.

What is ETL?

Three mismatched feeds become one clean table — watch each row get extracted, normalized to a shared shape, and loaded in.

Learn more

ETL is how messy data from many places becomes one tidy table. Extract pulls each feed in as-is, Transform rewrites every row into one shared shape (same column names, same units), and Load drops the clean rows into their final home.

What is a data platform?

The whole stack as one wiring diagram — connectors, bronze, silver, gold, backend, agent, app. Send a single coffee sale down the wires, block by block.

Learn more

A data platform is the full assembly line that carries one fact — say, a single coffee sale — from the app where it happened all the way to a dashboard or AI answer. Each stage (raw "bronze", cleaned "silver", business-ready "gold", then backend, agent, app) refines it a little more.

Who holds the stack?

Data engineer, data scientist, backend, frontend, full stack — pick a role and watch the slabs of the stack it owns light up.

Learn more

Software is built in layers, and different people own different layers. A frontend engineer owns what you see; a backend engineer owns the logic behind it; data engineers move and shape the data; data scientists find patterns in it; a full-stack engineer spans several layers.
